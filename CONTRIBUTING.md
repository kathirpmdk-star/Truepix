## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue with:
- Clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
- System information (OS, Python/Node versions)

### Suggesting Features

We welcome feature suggestions! Please:
- Check if the feature is already requested
- Clearly describe the use case
- Explain why it would benefit users
- Provide examples if possible

### Code Contributions

1. **Fork the Repository**
   ```bash
   git clone https://github.com/yourusername/truepix.git
   cd truepix
   ```

2. **Create a Branch**
   ```bash
   git checkout -b feature/amazing-feature
   ```

3. **Make Changes**
   - Follow existing code style
   - Add comments for complex logic
   - Update documentation if needed
   - Test your changes thoroughly

4. **Commit Changes**
   ```bash
   git add .
   git commit -m "Add amazing feature"
   ```

5. **Push to GitHub**
   ```bash
   git push origin feature/amazing-feature
   ```

6. **Open Pull Request**
   - Describe your changes clearly
   - Link related issues
   - Include screenshots for UI changes
   - Wait for review

## Development Guidelines

### Backend (Python)

- Follow PEP 8 style guide
- Use type hints where possible
- Write docstrings for functions
- Keep functions small and focused
- Handle errors gracefully

### Frontend (React)

- Use functional components
- Keep components small and reusable
- Use meaningful variable names
- Add PropTypes or TypeScript types
- Follow React best practices

### Model Development

- Document model architecture changes
- Include training metrics
- Test on diverse datasets
- Benchmark performance
- Explain any new techniques

## Testing

Before submitting:
- Test all endpoints (use `backend/test_api.py`)
- Verify UI works on different browsers
- Check mobile responsiveness
- Test with various image types
- Ensure no console errors

## Code of Conduct

- Be respectful and inclusive
- Help others learn
- Accept constructive feedback
- Focus on what's best for the project
- No harassment or discrimination

## Questions?

Feel free to:
- Open a discussion on GitHub
- Ask in pull request comments
- Reach out to maintainers
